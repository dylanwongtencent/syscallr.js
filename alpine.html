OpenX86 Alpine Linux in-browser target

Loads Alpine x86 minirootfs into the clean-room IA-32/Linux runtime, then starts BusyBox ash. Browser TCP uses the optional WebSocket gateway.
Start it with npm run gateway. Guest DNS is virtualized in the runtime; guest TCP uses the gateway’s /tcp endpoint.

A true CheerpX-equivalent Alpine session also requires complete IA-32, process, clone/vfork, signal, and dynamic-linker parity. This demo wires the Alpine rootfs and network path; unsupported opcodes/syscalls are printed when encountered.
